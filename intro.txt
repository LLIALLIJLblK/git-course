**Суммаризация страницы "Рабочий процесс | Введение в Git":**

- Страница описывает базовый рабочий процесс с Git: от инициализации репозитория до первого коммита.
- Основные этапы:
  - Инициализация репозитория с помощью команды `git init`.
  - Проверка статуса репозитория через `git status`.
  - Создание файлов проекта и отслеживание изменений.
  - Добавление новых или изменённых файлов в репозиторий через `git add`, после чего файлы попадают в индекс.
  - Окончательное сохранение изменений — коммит (`git commit -m "сообщение"`).
- Объясняется, почему процесс разделён на подготовку (`add`) и сохранение (`commit`): это позволяет формировать атомарные и логически завершённые коммиты.
- Приведён пример типового сценария работы (создание файла, статус, добавление в индекс, коммит).
- Рекомендуется делать осмысленные описания коммитов.
- Самостоятельное задание: установить Git, повторить шаги урока и добавить файл PEOPLE.md.
- Представлены полезные ссылки для дальнейшего изучения (Git cheatsheet, соглашение об именовании коммитов, официальная документация).

**Итого:**  
Страница предназначена для новичков, подробно описывает рабочий процесс с системами контроля версий на основе Git, начиная с создания проекта, добавления файлов и заканчивая первым коммитом, акцентируя внимание на принципах атомарности и осмысленности изменений.[1]
